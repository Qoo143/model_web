# 圖書館 RAG Agent 系統 - 教學文件目錄

## 文件架構說明

本系列教學文件從零開始，逐步教你建立一個完整的 RAG (Retrieval-Augmented Generation) 智能文件問答系統。每個文件都包含詳細的原理說明、實作步驟和程式碼範例。

## 學習路徑

### 📚 基礎篇

#### [01. 專案概述](01-project-overview.md)
**適合對象**: 所有人
**學習目標**: 了解專案整體架構、技術選型理由、核心功能
**關鍵概念**: RAG 是什麼、為什麼需要向量資料庫、群組權限設計

**你會學到**:
- RAG 系統的運作原理（用簡單例子說明）
- 為什麼選擇這些技術（Vue、FastAPI、Ollama、Chroma）
- 系統的核心流程（文件上傳 → 向量化 → 問答）
- 專案的目錄結構和各部分職責

### 🧠 核心技術篇

#### [04. RAG 基礎原理](04-rag-fundamentals.md)
**適合對象**: AI/ML 初學者
**學習目標**: 深入理解 RAG 的運作機制
**關鍵概念**: Embedding、向量相似度、語意搜尋

**你會學到**:
- 什麼是 Embedding（將文字轉成數字向量）
- 向量相似度如何計算（Cosine Similarity）
- 為什麼 RAG 比直接問 LLM 更準確
- Chroma 向量資料庫的運作原理
- BGE-M3 模型的特色（為什麼適合中文）
- 實際操作：體驗向量化和檢索過程

---

#### [05. 文件處理流程](05-document-processing.md)
**適合對象**: Python 初學者
**學習目標**: 學會解析和處理各種文件格式
**關鍵概念**: 文件解析、語意分塊、批次處理

**你會學到**:
- 如何解析 PDF、Word、Excel、Markdown
- 什麼是語意分塊（Semantic Chunking）
- 為什麼要用 chunk_overlap（重疊區域）
- 如何保留文件結構資訊（章節、頁碼）
- 批次處理大型文件的技巧
- 錯誤處理和狀態追蹤

---

#### [06. Ollama 與 LLM 整合](06-ollama-llm.md)
**適合對象**: LLM 初學者
**學習目標**: 學會在本地運行大型語言模型
**關鍵概念**: LLM、Prompt Engineering、本地部署

**你會學到**:
- 什麼是 Ollama（為什麼比 LM Studio 更適合）
- 如何下載和管理模型（gpt-oss-20b）
- 如何透過 API 與 Ollama 溝通
- Prompt Engineering 技巧（如何寫好提示詞）
- 如何將檢索到的上下文傳給 LLM
- 低耦合設計（未來可切換到 Gemini API）
- AMD GPU 加速設定

## 建議學習順序

### 🎯 快速入門（1-2 天）
如果你想快速看到成果：
1. 01 專案概述 → 02 Docker 環境 → 06 Ollama 整合
2. 啟動系統，先體驗完整功能

### 📖 完整學習（1-2 週）
如果你想深入理解每個技術：
1. **第一週**: 01 → 02 → 03 → 04
   - 理解整體架構和核心技術
2. **第二週**: 05 → 06 → 07 → 08
   - 實作具體功能

### 🔬 技術深入（依興趣選擇）
- **對 AI 有興趣**: 重點看 04（RAG 原理）、05（文件處理）、06（LLM）
- **對後端有興趣**: 重點看 03（資料庫）、07（認證）、08（FastAPI）
- **對部署有興趣**: 重點看 02（Docker）
